# 日本語のAI兆候カタログ

`humanize-prose` スキルのリファレンス。Step 2（兆候の検出）でチェックリストとして使う。

「兆候」とは、LLM 生成文に偏って現れるスタイル的特徴のこと。単発で出ても問題ではなく、**頻度**と**重なり**が AI らしさを生む。「いかがでしたか」を一回使うのは普通。それと「現代社会において」「〜することが重要です」「〜と言えるでしょう」が同じ記事に並ぶと、明らかに量産AI記事の臭いがする。

## 常套句・定型フレーズ

意味を運ばないが、文章の骨組みのように繰り返される。

| 兆候 | なぜAIっぽいか | 置換例 |
|------|----------------|--------|
| いかがでしたでしょうか / いかがでしたか | SEO量産AI記事の定番締め | 削除。問いかけで締めない |
| 〜することが重要です / 〜が大切です | 中身のない一般化ヘッジ | 何が重要かを直接書く（理由・対象・条件を明示） |
| 〜することをおすすめします | カジュアルAIブログの定型 | 推奨理由とともに具体動詞で（「〜してみるとよい」「〜するとXが得られる」） |
| ぜひ〜してみてください | 量産記事の定型呼びかけ | 削除、または具体的な手順だけ残す |
| 〜のではないでしょうか | 婉曲断定で逃げる | 言い切る、根拠を添える、または事実だけ書く |
| 〜と言えるでしょう / 〜と言えます | 出典なき断定の常套句 | 主体を明示する（「研究Xによれば〜」）、または削除 |
| 〜と言われています | 伝聞ヘッジ | 出典を示すか削除 |
| 結論から申し上げますと / 結論から言うと | 形式的すぎる導入 | 削除して結論を直接書く |
| 〜に他なりません | 強調定型句 | 削除、または「つまり〜だ」 |
| まさに〜である | 強調の常套句 | 削除 |
| 〜と考えられます | （技術文脈では正しいヘッジ） | 学術・技術文では残す。一般文ではしばしば削除可 |
| 一般的に〜とされています | 出典の隠蔽 | 出典を補うか、自分の観察として書き直す |
| 注目を集めています | 量産記事の常套句 | 削除、または具体的な事例で裏づける |
| 〜のひとつと言えます | 列挙ヘッジ | 削除、または「〜だ」と言い切る |
| 〜することで、〜できます | 取扱説明書ぽいAI構文 | 「〜すると〜になる」「〜すれば〜だ」 |

## 語尾・ヘッジ

文末処理に現れる癖。

| 兆候 | 置換例 |
|------|--------|
| 「〜です」「〜ます」の単調な連続 | 体言止め、断定（〜だ）、疑問の混在で抑揚をつける（カジュアル時） |
| 〜することができます | 「〜できる」 |
| 〜と思われます / 〜と思います（出典なし） | 主観として明示するか、根拠を補う |
| 〜となっています | 「〜だ」「〜である」 |
| 〜していただければと思います | 「〜してください」「〜してほしい」 |
| 〜と言っても過言ではありません | 削除、断定に置換 |

## 構文・並列

| 兆候 | なぜAIっぽいか | 置換 |
|------|----------------|------|
| 三点並列の多用（「AとBとC」「Xや、Yや、Z」「①〜、②〜、③〜」） | LLMの対称癖 | 二点 or 単独で十分なら減らす。三つ並べる必然性を確認 |
| 反射的な箇条書き | プロンプト出力癖。中身が連続したひと続きの説明なのに分割されている | 散文に戻す。本当に並列なものだけ箇条書きに残す |
| 文長の機械的均等 | 段落内の文長が10〜30字に揃いすぎる | 短文と長文を混ぜる |
| 段落ごとの長さが均一 | LLM出力の指紋 | 重要度に応じて長短を変える |
| 「メリット／デメリット」の対称提示 | 安全策としての両論併記 | 立場を取る。または非対称な実情を書く |
| 「まず〜、次に〜、最後に〜」 | 教科書スキーマの露出 | ナビゲーション語を削り、段落順で示す |

## 形式名詞・抽象化

| 兆候 | 置換 |
|------|------|
| 「〜こと」の多用（「〜することが」「〜であること」） | 具体動詞・名詞に置換（「〜することが重要」→「〜が要る」「〜せねばならない」） |
| 「〜もの」の多用 | 具体名詞に置換 |
| 「〜という」の連発 | 多くは削除可（「Xというもの」→「X」） |
| 「〜的」の多用（包括的、革新的、抜本的） | 具体的な性質を書く |
| 「〜化」（効率化、最適化、自動化） | 何がどう変わるかを書く |

## 開始（枕詞）

| 兆候 | 置換 |
|------|------|
| 現代社会において / 現代において | 削除、または時期と場面を具体化 |
| 近年 / ここ最近 / 昨今 | 年や具体的事象に置換 |
| 〜の重要性が高まっています | 何が高めているか、誰にとってかを書く |
| インターネット時代の今 / デジタル化が進む現代 | 削除、または具体的な変化を書く |
| 皆さんは〜したことはありますか | 削除（読者を測らない） |
| 〜という言葉を耳にしたことがあるでしょうか | 削除、語の説明を直接 |

## 終結

| 兆候 | 置換 |
|------|------|
| まとめ / 最後に / 以上のように | 削除。最後の論点で終わる |
| いかがでしたでしょうか | 削除 |
| 今後の動向に注目です / 注目していきたいですね | 削除、または具体的な観察ポイントを示す |
| ぜひ参考にしてください | 削除 |
| 一緒に〜していきましょう | 削除 |
| 本記事が〜の参考になれば幸いです | 削除 |

## 呼びかけ・読者誘導

| 兆候 | 置換 |
|------|------|
| 「皆さん」が定型的に文頭・文中に出る | 削除、または具体的な対象（「読者」「受講者」「初学者」）に絞る |
| 「あなた」が頻出（英語直訳調） | 文脈次第で削除、二人称が要らない場合は無人称化 |
| 「ぜひ」 | 削除 |
| 「〜してみましょう」（命令の婉曲形） | 「〜する」「〜してください」 |

## 句読点・記号

| 兆候 | 置換 |
|------|------|
| ダッシュ（—）を句読点代わりに連発 | 読点・句点・括弧で代用。ダッシュを使うなら一文書全体で1〜2回まで |
| 三点リーダー（…）の感情演出的多用 | ほとんど削除可。本当に省略する箇所だけに残す |
| 「！」「？」の多用 | 文末に断定で済ませる。感嘆符は強調が必要な箇所だけ |
| 全角・半角の混在の機械的整合（数字・括弧が常に半角→全角に変換されすぎる） | 媒体ルールに合わせて統一、ただし機械的すぎる場合は人間の判断で混在も可 |

## 段落・構成

| 兆候 | 置換 |
|------|------|
| 各段落が「主張＋根拠＋まとめ」で完結する反復 | 段落を主張だけで終わらせる、続く段落と内容を渡し合うなど構造を多様化 |
| 見出し（H2/H3）が等間隔に並ぶ | 重要度に応じて見出しの粒度を変える |
| 「タイトル → 導入 → 本論3章 → まとめ」のテンプレ構成 | 結論を先に置く、本論章数を不揃いにする、まとめを省く |

## 技術文書での注意

技術文・論文中の説明文では、以下を**AI兆候として扱わない**。これらは精度のための必要なヘッジ:

- 〜と考えられる / 〜と推測される / 〜と思われる（**根拠を伴うとき**）
- 〜の可能性がある / 〜とみられる（**観察事実に対する慎重表現**）
- 〜である場合がある（**条件付きの記述**）

AI 兆候として削るべきは、**根拠がなく内容も付加しない**ヘッジ（一般文での「〜と言えるでしょう」「〜のではないでしょうか」など）。

## 頻度メトリクス（量で判定する目安）

以下に該当する場合、AI兆候の濃度が高い:

- 1記事に「〜することが重要です」「〜と言えるでしょう」「いかがでしたか」のいずれかが2回以上
- 三点並列が1段落に2回以上
- 「現代社会において」「近年」を含む段落が冒頭・中盤・末尾に3回以上
- 段落長の標準偏差が小さい（5段落以上で長さが揃う）
- 箇条書きの全項目が同じ長さ・同じ構文
- 形式名詞「〜こと」が1段落に5回以上

これらが積み重なっているときは、書き換え強度を上げる。
